Cribs — Technical Spec for Replit Agent
What to build
A mobile-first social media web app called Cribs where every post is a real estate listing. Users scroll a feed of homes (active, sold, price-reduced), react with house-specific emoji reactions, comment, and explore a neighborhood heatmap. Think Instagram meets Zillow meets r/ZillowGoneWild.
The app should feel like a social media app first, real estate tool second. Entertainment and commentary drive engagement, not search intent.

Tech stack

Frontend: Next.js with React, Tailwind CSS
Backend/Database: Supabase (auth, Postgres database, real-time subscriptions)
Listing Data: SimplyRETS demo API (credentials: username simplyrets, password simplyrets, endpoint https://api.simplyrets.com/properties)
Maps: Mapbox GL JS (free tier) for heatmap view
Image optimization: Next.js Image component with SimplyRETS photo URLs
Deployment: Replit hosting


Brand

Name: Cribs
Logo: A simple house icon inside a rounded red (#E24B4A) square, with "cribs" in lowercase next to it
Vibe: Clean, modern, mobile-native. White cards, full-bleed photos, bold prices. The design should feel like Instagram — not like a real estate search portal
Font: System font stack (Inter or similar sans-serif)


Pages and features
1. Feed (home page, /)
A vertical infinite-scroll feed of listing cards. Each card contains:
Photo section:

Full-width hero image from the listing photos array
Swipeable photo carousel (left/right) with dot indicators
Photo count badge in top-right corner (e.g., "1 / 24")

Price overlay (bottom of photo, over a subtle dark gradient):

Price status label in smaller text above the price:

"Just dropped" — new/active listing (status: Active)
"Gone for" — sold listing (status: Closed), show sale price large, list price struck through below
"Slashed -$Xk" — listing with price reduction, show current price large, original price struck through below
"Pending" — under contract (status: Pending)
"Back from the dead" — listing that was pending and came back to active


Price in large bold white text (e.g., "$1,250,000")
Value badge in bottom-right corner of the photo (colored pill):

Green (#1D9E75) "Gem" — listing $/sqft is more than 15% below neighborhood average
No badge — within ±15% of average (don't show "Fair", just omit)
Amber (#BA7517) "Sus" — 15-30% above neighborhood average
Red (#E24B4A) "Delusional" — more than 30% above neighborhood average

Street address (bold, 16px)

Reaction bar:

Six reaction buttons in a row, evenly spaced, with emoji above and count below:

🔥 (fire / dream home)
🤮 (gross / nightmare)
💀 (WTF)
💰 (flex / money)

💬 (comment — opens comment thread)
Share icon (SVG arrow, opens native share sheet or copy link)


Tapping a reaction toggles it (like/unlike behavior). A user can react with multiple reactions on the same listing.
Reaction counts update in real-time via Supabase realtime subscriptions

Hashtags:

Users can add custom hashtags in comments (anything prefixed with #)

Comment preview:

"View all X comments" link below in muted text

Feed tabs (horizontal scrollable pills at top of feed):

"For you" (default — all listings sorted by engagement/recency)

Database views
listing_feed — a view that joins listings with aggregated reaction counts:
sqlSELECT 
  l.*,
  COUNT(r.id) FILTER (WHERE r.reaction_type = 'fire') as fire_count,
  COUNT(r.id) FILTER (WHERE r.reaction_type = 'gross') as gross_count,
  COUNT(r.id) FILTER (WHERE r.reaction_type = 'wtf') as wtf_count,
  COUNT(r.id) FILTER (WHERE r.reaction_type = 'flex') as flex_count,
  COUNT(r.id) FILTER (WHERE r.reaction_type = 'gem') as gem_count,
  COUNT(c.id) as comment_count
FROM listings l
LEFT JOIN reactions r ON r.listing_id = l.id
LEFT JOIN comments c ON c.listing_id = l.id
GROUP BY l.id

Data ingestion (SimplyRETS sync)
Create a server-side function (Next.js API route or Supabase Edge Function) that:

Calls GET https://api.simplyrets.com/properties with auth simplyrets:simplyrets
Paginates through all results (use limit=500&lastId=0, follow the Link header for next page)
For each listing, upsert into the listings table (match on mls_id)
Compute price_per_sqft = price / sqft
Compute area_avg_price_per_sqft — average $/sqft of all listings within the same city
Compute value_badge based on how the listing's $/sqft compares to area average:

More than 15% below → "gem"
Within ±15% → "fair"
15-30% above → "sus"
More than 30% above → "delusional"


Auto-generate hashtags based on property data (property type, features mentioned in description, price tier)
Run this sync on app startup and then every 6 hours (or manually triggered)

Responsive behavior

Mobile (<480px): single column feed, full-width cards
Tablet (480-1024px): single column feed, centered with max-width
Desktop (>1024px): single column feed centered, max-width 480px, with map view optionally showing as a sidebar

The app should feel like a mobile app even on desktop. Don't stretch the feed to fill wide screens.

Key interactions
Scrolling the feed: Infinite scroll, load 10 listings at a time, skeleton loading cards while fetching.
Reacting: Tap an emoji reaction to toggle it. Immediately update the count optimistically (before server confirms). Show a brief scale-up animation on the emoji when tapped.
Commenting: Tap the comment icon or "View all comments" to open the comment section. Text input at the bottom with send button. New comments appear immediately (optimistic update). Replying to a comment indents one level.
Saving: Tap the bookmark icon on any card. Heart fills in. Optionally prompts to add to a collection.
Sharing: Tap share icon, opens native share sheet (or copies link on desktop).
Map interaction: Pan and zoom freely. Pins cluster at low zoom levels, spread out as you zoom in. Tap a pin to see the mini-card popup. Tap the mini-card to navigate to the listing detail.
Photo carousel: Swipe left/right on the listing photo to browse all photos. Dot indicators show position. Tap to open fullscreen gallery.

What NOT to build in V1

User-to-user messaging / DMs
Agent accounts or listing management
Mortgage calculator
Advanced search filters beyond neighborhood/hashtag
Push notifications
Native mobile app (web-only for now)
Sophisticated valuation model beyond simple $/sqft comps
Permit/assessor data cross-referencing
User-uploaded content (only MLS listings)
Moderation tooling beyond basic report button
Analytics dashboard
Monetization features


Success criteria for the prototype

Feed loads with real listing data from SimplyRETS
Cards display photos, prices, status labels, stats, value badges, and comp context
Users can sign up, react, comment, and save
Comments are threaded and voteable
Map view shows listing pins with value color-coding
Feed tabs filter correctly (Gone Wild, Just Dropped, Gone, etc.)
The app looks and feels like a polished social media app, not a real estate portal
